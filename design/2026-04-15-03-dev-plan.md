# TT ShadUni Skill 开发计划

> 基于设计文档 v3（`2026-04-15-skill-design.md`）

## 概览

| 项目 | 说明 |
|------|------|
| 目标 | 构建可用的 TT ShadUni AI Skill |
| 输入 | tt-shaduni（65 组件，63 有 props.ts）+ tt-daka-mp（参考架构） |
| 输出 | SKILL.md + knowledge/ + templates/ + scripts/ |
| 分 5 个 Phase | 每个 Phase 独立可验证 |

## Phase 1：核心入口（SKILL.md + overview）

**目标**：完成 Layer 0，使 Skill 可被 Cursor 加载并识别工作模式。

| # | 任务 | 输出文件 | 依赖 |
|---|------|---------|------|
| 1.1 | 编写 SKILL.md（≤100 行） | `SKILL.md` | 无 |
| 1.2 | 编写项目概览 | `knowledge/01-overview.md` | 无 |
| 1.3 | 生成 AGENTS.md / CLAUDE.md | `AGENTS.md`, `CLAUDE.md` | 1.1 |

**1.1 SKILL.md 内容清单**（按设计文档 Section 8）：
- 概览（5 行）：包名、风格、框架、组件前缀、图标、云服务、平台
- 工作模式（15 行）：查询/生成/修改模式判断规则
- 代码规范（10 条）
- 组件分类索引（6 个分类，只列组件名）
- 知识文件路由表

**1.2 overview 内容清单**：
- 技术栈：Vue3 + UniApp + Pinia + EMAS Serverless
- 项目结构：tt-shaduni（组件库）+ tt-daka-mp（参考实现）
- 构建产物：H5 / 微信小程序 / 支付宝小程序
- 组件系统：easycom 自动注册，tt- 前缀
- 样式系统：CSS 变量 + rpx + BEM

**验证**：Cursor 中打开 tt-shaduni-skill 目录，确认 Skill 出现在 available_skills 列表中。

## Phase 2：组件知识（脚本 + 6 个分类文件）

**目标**：自动生成 63 个组件的 API 文档，按 6 个分类组织。

| # | 任务 | 输出文件 | 依赖 |
|---|------|---------|------|
| 2.1 | 编写 gen-component-docs.mjs | `scripts/gen-component-docs.mjs` | 无 |
| 2.2 | 运行脚本生成 6 个分类文件 | `knowledge/components/*.md` | 2.1 |
| 2.3 | 人工审查 + 补充 events/slots/示例 | 同上 | 2.2 |

**2.1 脚本设计**：

```
输入: tt-shaduni/packages/components/*/props.ts（63 个文件）
处理:
  1. 读取每个 props.ts，解析 TypeScript 定义
  2. 提取: prop 名、类型、默认值、描述（从注释）
  3. 按预定义分类映射分组:
     - basic: tt-button, tt-icon, tt-image, tt-typography, tt-space, tt-divider
     - layout: tt-row, tt-col, tt-grid, tt-safe-area, tt-sticky, tt-scroll-view
     - navigation: tt-navbar, tt-tabbar, tt-tabs, tt-sidebar, tt-breadcrumb, tt-index-bar, tt-pagination, tt-steps
     - form: tt-input, tt-textarea, tt-switch, tt-checkbox, tt-radio, tt-slider, tt-rate, tt-number-box, tt-search, tt-picker, tt-date-picker, tt-upload, tt-form, tt-form-item
     - display: tt-cell, tt-card, tt-tag, tt-badge, tt-avatar, tt-list, tt-table, tt-descriptions, tt-collapse, tt-count-down, tt-progress, tt-swiper, tt-calendar, tt-drag
     - feedback: tt-dialog, tt-popup, tt-sheet, tt-action-sheet, tt-toast, tt-loading, tt-skeleton, tt-empty, tt-notice-bar, tt-tooltip, tt-dropdown-menu, tt-swipe-action, tt-transition, tt-config-provider
  4. 每个组件生成 props 表格 + 源码路径引用
输出: knowledge/components/{basic|layout|navigation|form|display|feedback}.md
```

**降级策略**：所有 63 个组件均有 `props.ts`，无需降级。但脚本仍保留 `.vue` 文件扫描作为保险。

**验证**：检查生成的 6 个文件，每个组件都有 props 表格和源码路径。

## Phase 5：验证与调优

**目标**：端到端验证 Skill 的两种工作模式，确保渐进式披露正常工作。

| # | 测试场景 | 预期行为 |
|---|---------|---------|
| 5.1 | 查询: "tt-button 有哪些 variant" | 加载 components/basic.md → 返回 props 表格 |
| 5.2 | 查询: "EMAS 怎么连接数据库" | 加载 06-cloud-emas.md → 返回 EMAS 用法 |
| 5.3 | 查询: "项目的样式系统怎么用" | 加载 04-theme.md → 返回 CSS 变量说明 |
| 5.4 | 生成: "生成一个任务管理模块" | 展示字段映射预览 → 生成 8 类文件 |
| 5.5 | 修改: "给 project 加一个搜索功能" | 先读现有代码 → 增量修改 |
| 5.6 | 跨模式: "tt-calendar 怎么用？帮我做个日程页面" | 先查询 → 再生成 |
| 5.7 | Layer 2 深入: "tt-calendar 的 formatter prop 具体怎么用" | 加载 display.md → 需要精确信息 → 读源码 props.ts |

**调优方向**：
- SKILL.md 的模式判断规则是否准确
- 知识路由是否将请求导向正确的文件
- 生成代码是否符合 tt-daka-mp 的架构模式
- 组件选择是否合理
